RDF Feature Survey

Ontology and Vocabulary Management

1. Predefined Templates for Common RDF Vocabularies ▼

Description: This feature provides ready-to-use templates for popular RDF vocabularies, including FOAF, Schema.org, DC, etc... . When selected, the template inserts structured triples into the RDF document, defining entities with properties commonly used in these vocabularies. For example, a template for a person might include basic information such as foaf:name, foaf:age, and schema:address, with slots for values like name, email, and address details already laid out.

Why Useful: This feature is a time-saver, especially for users who frequently work with common vocabularies. Instead of typing each term manually, users can start with a structured set of triples that reflect the vocabulary's typical usage. For beginners, it offers a concrete example of how to apply RDF vocabularies correctly. For experienced users, it ensures faster setup and minimizes errors, helping to keep data consistent and standardized.

Supported Serialization: Turtle, JSON-LD

Usage in VS Code: Available through the command palette, allowing users select and insert available templates into RDF documents.

Using template

2. User-Defined Custom Templates ▼

Description: This feature allows users to create, name, and remove their own RDF templates, tailored to their specific data needs. Users can define any custom structure by setting up triples with their preferred properties, namespaces, and values, then save this setup as a reusable template. Once created, templates can be easily inserted into any RDF document with a single command. Users can also remove templates, keeping their template library organized and relevant to their ongoing projects.

Why Useful: Custom templates are invaluable for users with unique data requirements that aren't fully covered by predefined templates. By allowing users to create templates tailored to their specific data structures, this feature prevents the need to recreate complex setups repeatedly. Users can build templates once and reuse them across projects, ensuring consistency and saving time.

Supported Serialization: Turtle, JSON-LD

Usage in VS Code: Users name and save, as well as remove templates via command palette, then users are able to select and insert the templates through command palette.

3. RDF Dataset Diff Viewer ▼

Description: The RDF Dataset Diff Viewer allows users to instantly compare two RDF datasets side-by-side, highlighting overlapping and similar triples in each dataset. Users can see which triples are present in only one dataset and examine detailed differences within similar triples, providing a clear view of any discrepancies. The triples don't need to align in both of the datasets.

Why Useful: This feature is ideal for users who need to compare two datasets directly, especially when working with data from different sources or independently modified files. By showing similar triples, the Diff Viewer helps users quickly assess consistency, compatibility, and potential conflicts between datasets.

Supported Serialization: Turtle, N-Triples

Usage in VS Code: Users can access the Diff Viewer via the command palette, selecting RDF file for comparison against the current open file. Differences are highlighted in a side-by-side view, with similar triples marked and on click of this triples we get to scroll automatically to the similar triple on the other file.

Diff Viewer

4. RDF Dataset Merger with Conflict Resolution ▼

Description: The RDF Dataset Merger enables users to merge two RDF datasets by identifying and resolving conflicting triples. In RDF, conflicts can arise in several ways, such as namespace clashes, semantic inconsistencies, or incompatible datatypes. Users can view each conflicting entry (just like in number 3 - "Diff Viewer") and choose which version to retain. This process ensures that only the most relevant and accurate triples are included in the merged output, which can then be saved as a new RDF file.

Examples:

  • Namespace Clashes: When two datasets define the same prefix for different namespaces:
    @prefix ex: <http://example.org/> . (Dataset 1)
    @prefix ex: <http://another-example.org/> . (Dataset 2)
  • Semantic Inconsistencies: When the same entity is described with contradictory properties:
    <ex:Homer> <ex:age> "34"^^xsd:integer . (Dataset 1)
    <ex:Homer> <ex:age> "43"^^xsd:integer . (Dataset 2)

Why Useful: Merging datasets with conflict resolution allows users to integrate data from different sources without creating duplicates or inconsistencies. This feature is particularly useful in collaborative environments, where changes from multiple contributors need to be combined.

Supported Serialization: Turtle

Usage in VS Code: Users initiate the merge through the command palette, selecting RDF file for comparison against the current open file; and review conflicts in a conflict-resolution view (a side-by-side view). This would be similar to GIT, we see the conflicts like in the "Diff Viewer" in number 3 above; after deciding on resolutions which to keep/discard, modify or merge triples, the final merged dataset can be saved as a new RDF file.

5. Autocomplete with Support for Custom Vocabularies ▼

Description: This feature combines autocompletion with support for custom vocabularies, allowing users to seamlessly integrate standard and user-defined RDF terms. As users type, the autocomplete feature suggests vocabulary terms for both predefined and custom vocabularies. If a prefix is referenced but not yet imported, it automatically inserts the appropriate @prefix declaration. Additionally, users can load custom vocabulary files, enabling autocompletion for terms specific to their projects.

Why Useful: This feature allows users to reference both well-known and custom vocabularies without manually typing terms or setting up prefixes. It is especially useful for projects that require unique vocabularies or project-specific terms, saving time and reducing errors.

Supported Serialization: Turtle, JSON-LD

Usage in VS Code: Users can configure custom vocabulary files in settings by specifying file or folder. As users type terms from both standard and custom vocabularies are suggested. Additionally, the autocompletion feature automatically handles prefix imports as needed.

Data Validation and Quality Assurance

6. Comprehensive Linting, Validation, and Formatting for RDF Data ▼

Description: This feature provides thorough linting, validation, and automatic formatting, ensuring compliance with best practices while maintaining readability. It detects and highlights a variety of common issues, such as unknown vocabularies, missing language tags or value types for literals, and data mismatches. The integrated Prettier-style formatter applies consistent spacing, line breaks, and indentation, and also allows customization of these formatting rules. For quick fixes, users can utilize syntax shortcuts and apply suggested corrections with a single click, improving data quality and productivity. Additionally, users can selectively ignore certain warnings to focus on priority issues.

Why Useful: This feature allows users to both detect and resolve issues promptly, while ensuring their RDF data remains organized and easy to read. The ability to apply fixes and ignore specific warnings makes it adaptable for both beginners, who benefit from immediate feedback, and advanced users, who can tailor the tool to their specific needs.

Supported Serialization: Turtle

Usage in VS Code: Issues are highlighted as users type, with suggestions and one-click fixes available on hover. Users can access formatting options via right-click or the command palette to style the entire document or selected sections. Warnings can be ignored on a case-by-case basis, and custom formatting options can be configured in settings.

7. JSON-LD Validation ▼

Description: This feature verifies that property values match expected data types (e.g., names as strings, ages as integers) and issues warnings for missing required properties, duplicate @id values, and duplicate keys. Optional alerts are provided for missing @type and @id fields to enhance interoperability.
Additional checks focus on @context usage: they ensure consistency between property names and their definitions in the @context, validate correct usage of @reverse and @nest, confirm that terms in the @context map correctly to IRIs, and detect invalid nesting of @context definitions. Further validation ensures adherence to IRI/URL formatting standards, appropriate vocabulary usage, and compliance of @language tags with ISO standards.

Why Useful: This feature ensures data consistency and semantic accuracy in JSON-LD files, avoiding common issues like mismatched data types or misconfigured contexts. By flagging potential errors and inconsistencies, it helps maintain high standards for structured data and saves users from manually checking JSON-LD specifications

Supported Serialization: JSON-LD

Usage in VS Code: Users see real time validation as they type or edit JSON-LD content. Warnings and errors are highlighted in-line, with tooltips providing details about the issue and suggested fixes.

JSON-LD Validation

8. Warn Usage of Duplicate Triples ▼

Description: This feature detects duplicate triples within an RDF file, identifying redundant data entries even when they appear in different forms, such as using prefixed terms (e.g., ex:name) versus full IRIs (e.g., http://example.org/name). By flagging these duplicates, the tool helps users maintain data integrity and prevent data bloat, ensuring an optimized and consistent file structure.

Why Useful: Duplicate triples can inflate file size and create confusion. By flagging duplicates, this feature helps keep datasets clean and optimized, promoting efficient RDF writing practices and readability, especially in large datasets.

Supported Serialization: Turtle, N-Triples

Usage in VS Code: Duplicate triples are highlighted in the open dataset.

Formatting and Organization

9. Sort and Group RDF Triples by Entity ▼

Description: This feature organizes RDF triples by grouping them according to individual entities, ensuring that all triples related to the same entity are displayed together. For example, all triples associated with a person (such as name, age, and email) will be clustered in one place, providing a clear, concise view of that entity's properties and relationships.

Why Useful: In RDF datasets with complex interconnections, triples related to the same entity can become scattered, making it challenging to follow information about a particular instance. Grouping triples by entity enhances readability, allowing users to view all relevant data for each individual entity without searching through unrelated triples.

Supported Serialization: Turtle, N-Triples

Usage in VS Code: Users can activate entity-based sorting through the command palette, which applies the grouping across the RDF document, displaying each entity's triples in a unified block.

10. Hide Long URLs for Improved Document Structure ▼

Description: This feature hides long URLs in RDF documents to improve readability and orientation, replacing them with shorter, user-friendly versions (eg. http://example.org/...). This visual adjustment is purely for display, allowing users to view a version of their data without altering the actual URIs. Users can toggle the visibility of long URLs, making it easy to switch between a clean view and the original format as needed.

Why Useful: RDF documents often contain lengthy URIs, which can clutter the workspace and make it challenging to follow the document's structure. By hiding these long URLs, this feature helps users focus on the data content itself, improving orientation and reducing visual noise.

Supported Serialization: Turtle, JSON-LD

Usage in VS Code: Users can toggle this option, hiding URLs while retaining their functionality, also revert back to the original long url, through a mouse or command palette

11. Batch IRI/Prefix Refactoring ▼

Description: This feature enables bulk refactoring of IRIs and prefixes across RDF datasets, allowing users to switch between full IRIs and shortened prefixes in a single operation. For example, users can convert http://example.org/name to ex:name (and vice versa) across current document or entire datasets. This makes it easy to adapt the RDF data structure to project needs, improving readability and consistency.

Why Useful: In RDF data, balancing between full IRIs and prefixes can be crucial for readability and consistency, especially in large datasets. Manually converting each IRI to a prefix (or vice versa) is time-consuming and prone to error. By automating these bulk edits, it reduces repetitive work, minimizes formatting errors.

Supported Serialization: Turtle, JSON-LD

Usage in VS Code: Users can select IRI or prefix notation, then activate refactoring across RDF datasets.

Data Exploration, and Visualization

12. Interactive Graph/Visualization for RDF ▼

Description: This feature provides an interactive graph-based visualization of RDF data, allowing users to explore entities and their relationships as interconnected nodes. Users can switch between schema views, which display the structural hierarchy of classes and properties, and instance views, which show specific data entries and their links. Similar to existing tools like WebVOWL, this feature makes it easy to understand complex RDF structures by visually mapping out connections and dependencies.

Why Useful: RDF data often involves complex, nested relationships that are hard to interpret in plain text. An interactive graph simplifies exploration, offering a high-level view and detailed entity connections, enhancing understanding and data accessibility for all users.

Supported Serialization: Turtle, JSON-LD

Usage in VS Code: Users open the graph in a side panel, clicking on nodes to explore related entities.

13. Dataset Overview: VoID & Statistical Analysis ▼

Description: This combined feature provides a comprehensive overview of RDF datasets by offering users the choice to view either a VoID summary or statistical metrics. The VoID summary captures the dataset structure and interlinking relationships, complemented by the statistical metrics to provide insights such as centrality, connectivity, and node significance. By extracting and presenting these insights together, users can choose the view that best suits their analysis needs.

Why Useful: The VoID summary aids in understanding dataset metadata for easier sharing and integration, while statistical metrics offer insights into structural complexity and significant entities. Together, they provide options for high-level and detailed dataset analysis.

Supported Serialization: Turtle, JSON-LD, N-Triples

Usage in VS Code: Through the command palette, users can select to generate a VoID summary or view statistical metrics. The chosen overview is displayed in a new tab for easy review.

14. Allow Filtering of RDF Triples ▼

Description: This feature enables users to filter RDF triples based on specific criteria such as subject, predicate, object, or namespace. By narrowing down the displayed triples, users can focus on particular section of the dataset, making it easier to examine relevant relationships or data points.

Why Useful: Large RDF datasets can be overwhelming to navigate, especially when looking for specific patterns or subsets of data. The ability to filter triples by subject, predicate, object, or namespace offers an approach to data exploration, allowing users to locate relevant information without needing to look through unrelated data.

Supported Serialization: Turtle, JSON-LD
